## 일자별 진행 사항
### 2026.01.12 (월)

 - 팀원 각자가 사전에 준비한 아이디어를 공유하고, 아이디어별로 강점과 한계점을 중심으로 논의

 - 기존 아이디어에 대해 피드백을 반영하며 방향성 보완

 - 최종 아이디어 선별을 위한 기준(실현 가능성, 차별성, 확장성 등)을 설정하고 회의 진행

### 2026.01.13 (화)

 - 협업을 위한 개발 규칙 회의 진행

    - Git 커밋 컨벤션 정의

    - 브랜치 전략 및 PR 규칙 정리

 - 팀을 3:3으로 나누어 전날 선별된 2개의 기획안에 대해 추가 자료 조사 진행

 - 조사 결과를 공유한 뒤 비교·논의를 통해 최종 기획안 채택

 - 최종 기획안을 기준으로 팀 내 포지션 결정

### 2026.01.14 (수)

 - 프로젝트 진행을 위한 교보재 수령 및 정상 작동 여부 확인

 - 기존 기획안을 기반으로 추가 기능 브레인스토밍 진행

 - 서비스 사용 흐름을 중심으로 사용자 시나리오 회의 진행

    - 주요 사용자

    - 사용 상황

    - 기능 간 흐름 정리

### 2026.01.15 (목)

 - 전체 기획안에 대한 내부 컨펌 진행

 - 컨펌 과정에서 도출된 피드백을 정리하고 반영 방향 논의

 - 확정된 방향을 기준으로 기능 명세서 작성 시작

    - 핵심 기능 정의

    - 기능별 목적 및 역할 정리

### 완료 사항
- 팀원별 아이디어 공유 및 논의를 통해 프로젝트 문제 영역을 구체화
- 기획안 선별 기준을 수립하고, 최종 프로젝트 기획안을 확정
- Git 커밋 컨벤션, 브랜치 전략, PR 규칙 등 협업을 위한 개발 규칙을 정리
- 사용자 흐름을 중심으로 서비스 사용 시나리오를 정리
- 기획 컨펌을 통해 프로젝트 방향성을 확정하고 기능 명세서 작성

### 진행 예정
- 기능별 역할 분담 및 담당자 확정
- 요구사항 명세서 및 기본 설계 문서 작성
- 개발 환경 세팅 및 초기 프로젝트 구조 구성
- 기능 구현을 위한 개발 단계 착수
