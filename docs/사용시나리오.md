# 사용 시나리오
### [앱 초기 진입 및 설정]

- 상황 : 보호자가 독거 중인 어머니/아버지를 위해 기기를 설치하고 앱을 세팅하는 과정
1. 회원 가입 및 로그인 (OAuth 고려)
2. 가족 구성원 초대 및 그룹 생성
    - 그룹명 설정
    - “초대하기” 버튼을 눌러 가족 단톡방에 초대 링크를 보냄
    - 초대받은 가족들이 ‘가족 멤버’로 등록됨
    - 역할 설정 - 부양자 or 피부양자
3. AIoT 기기 연결 및 환경 테스트
    - 피부양자 기준: 앱에서 “기기 등록”을 누르고, AIoT의 QR코드를 스캔하여 Wi-Fi 연결
    - AIoT 기기 위치 설정 및 설치
    - 웨어러블(스마트 워치) 연동
4. 사용자 프로필 및 안전 가이드 설정
    - 기본 정보 : 어머니/아버지의 생년월일, 혈액형, 기저 질환(예: 고혈압) 입력
    - 응급 연락망 설정
        - ex) 1순위 - 큰아들, 2순위 - 작은 딸, 3순위 - 옆집 이웃, 4순위 - 119 안전신고센터
    - 목소리 학습(TTS) : 가족들의 음성 학습

### [위험 상황 발생 시나리오 A : 낙상 사고(High Risk)]

- 상황 : 거실에서 이동 중이던 피부양자가 미끄러져 넘어지고, 의식을 잃은 위급 상황
1. AIoT 위험 상황 인식
    - 영상 분석 : 카메라가 스켈레톤(관절 포인트)이 급격히 하강(y축 변화)하고, 바닥에 눕는 형태(Pose : Lying)로 변한 것을 감지
    - 1차 판단 : “낙상 의심 상황”으로 분류하고 즉시 서버로 이벤트 트리거 전송
2. 데이터 요청
    - 폰 앱에 심박수 데이터 요청
    - 폰 앱에서는 저장된 심박수 데이터 서버로 전송
3. 현장 1차 대응
    - 음성 확인 : 괜찮은지 묻는 음성 출력
    - 무응답 감지
4. 보호자 알림 및 행동 가이드
    - 긴급 알림 : 설정된 응급 연락망의 스마트폰에 푸시 알림이 뜸 (심박수 이상 시 긴급 설정)
    - 상황 파악 : 알림을 누르면 위험상황(낙상, 온열질환, 심정지 등)과 현재 심박수 데이터 뜸
    - 행동 가이드 실행 : 화면 하단에 단계별 버튼이 활성화됨
5. 사후 기록(Log)
    - 상황 종료 후, 시스템은 위험 상황 로그에 자동 기록
        - 발생 일시, 원인, 조치 등

### [위험 상황 발생 시나리오 B : 온열 질환 경고 (Medium Risk)]

- 워치로 파악해서 폰으로 보냄

### [사진 디스플레이 기능]

- 컨셉: 까만 화면의 기계가 아니라, 늘 가족사진이 흐르는 따뜻한 액자가 거실에 놓여 있습니다.

1. 평상시 상태

- 화면: 라즈베리파이에 연결된 디스플레이에서 가족사진이 슬라이드 쇼로 천천히 넘어갑니다.
- 클라우드 동기화: 자녀들이 각자의 폰에서 앱에 사진을 올리면, 1분 뒤 할머니 댁 액자(기기) 구석에 "새 사진 도착!" 알림이 뜨고 슬라이드 쇼 목록에 자동 추가됩니다.

2. 인터랙션 (Touch & Feel) (나중에 추가 기능)

- 터치: 할머니가 지나가다 손주 사진을 보고 화면(혹은 터치센서)을 톡 건드리면, 사진이 멈추고 전체 화면으로 확대됩니다.
- 좋아요: 한 번 더 길게 누르면, 자녀의 앱에 "할머니가 [졸업식 사진]을 좋아하십니다 ❤️" 알림이 갑니다.

3. 이벤트 발생 시 (Interrupt)

- 알림 모드 전환: 약 먹을 시간이 되거나 문이 열리면, 사진이 흐려지고(Dimming) 중앙에 "💊 약 드실 시간이에요" 텍스트와 아이콘이 크게 뜹니다.
- 위험 상황: 낙상이 감지되면 즉시 사진이 꺼지고 빨간색 경고 화면(또는 영상 통화 대기 화면)으로 바뀝니다.

## [캘린더 기능]

### 시나리오 1: 가족 방문 기록 & 예정 관리

1. 보호자가 앱에서
    
    → “방문 일정 추가”
    
    - 날짜/시간
    - 방문자 (아들, 딸 등)
    - 메모 (병원 동행, 식사 등)
2. 캘린더에 **“방문 예정”**으로 표시됨
3. 방문 당일:
    - 독거 노인에게 음성 알림
        
        > “오늘 오후 3시에 ○○님이 방문 예정입니다.”
        > 
4. 방문 완료 후:
    - 보호자가 ‘방문 완료’ 체크
    - 캘린더에 방문 이력으로 저장
